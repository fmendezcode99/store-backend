# Store Backend - Project Context

## Overview

Store Backend is a long-term backend engineering project built to learn professional software development while creating a production-quality portfolio.

This repository is not intended to be a quick prototype. Every architectural decision, implementation, and document should contribute to building maintainable, scalable, and well-documented software.

---

# Objectives

The project has four primary objectives:

1. Learn backend engineering through real-world development.
2. Build a professional portfolio project.
3. Apply software engineering best practices.
4. Continuously improve code quality and technical knowledge.

Learning is as important as delivering working code.

---

# Development Philosophy

The project follows these principles:

- Learn before optimizing.
- Build incrementally.
- Keep the architecture simple.
- Prefer readability over clever solutions.
- Document important decisions.
- Write code that is easy to maintain.
- Favor long-term quality over short-term speed.

---

# Development Workflow

The general workflow is:

1. Understand the problem.
2. Design a solution.
3. Implement the smallest useful change.
4. Test the implementation.
5. Review the code.
6. Update documentation when necessary.
7. The developer decides when to commit.

The AI assists throughout the process but never controls Git operations.

---

# AI Responsibilities

The AI should:

- Explain concepts before implementing.
- Respect the existing architecture.
- Recommend best practices.
- Review code constructively.
- Encourage independent learning.
- Ask questions when requirements are unclear.

The AI is a collaborator, not an autonomous developer.

---

# Quality Standards

Every contribution should improve at least one of the following:

- Readability
- Maintainability
- Simplicity
- Consistency
- Documentation
- Developer understanding

Working code is important, but understandable code is the real objective.

---

# Long-Term Vision

The long-term goal is to transform Store Backend into a complete backend application that demonstrates professional software engineering practices.

The repository should eventually include:

- Clean architecture
- Database migrations
- REST API
- Authentication
- Automated testing
- Logging
- Configuration management
- Deployment
- Continuous Integration
- Complete documentation

The project should demonstrate both technical competence and professional development practices.

---

# Guiding Principle

Every task completed in this repository should leave the project in a better state than it was before.

Small, consistent improvements are preferred over large, disruptive changes.
