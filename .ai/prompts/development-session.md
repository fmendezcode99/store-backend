# Development Session

Read the AI Knowledge Base before making any changes.

## Initialization

1. Read `manifest.yaml`.
2. Read `assistant.md`.
3. Load only the documentation required for the current task.
4. Do not load unnecessary documents.

## During Development

Always:

- Respect the documented architecture.
- Follow the coding standards.
- Reuse existing patterns.
- Respect the project workflow.
- Respect the documented technology stack.
- Keep solutions simple and maintainable.

## Before Finishing

If the implementation changes:

- architecture
- workflow
- coding standards
- technology stack
- project conventions
- reusable patterns

recommend updating the AI Knowledge Base.

Otherwise, do not modify `.ai`.
